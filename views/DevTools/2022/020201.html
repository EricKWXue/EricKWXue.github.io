      NAS安装Clash代理

      Eric Xue 2022-02-02 NAS

      # NAS安装Clash代理

      # 1. 需求

      买了日版的switch，就会在玩游戏的道路上碰到很多问题，比如购买游戏啊、下载游戏啊、更新游戏啊...之类的都需要访问远在海外网站。 如果没有代理，遇到以上问题，真的寸步难行！

      # 2. 规划

      用家里的NAS（威联通）专门设立一个Clash-Proxy的虚拟机（docker）作为代理接入点，凡是需要访问外网的都走这条代理通道。

      # 3. 安装

      # 3.1 准备

      在威联通的Container Station所在目录下新建两个目录

      /share/Container/clash/config    #存放clash配置文件
/share/Container/clash/ui        #存放控制面板UI
      1
      2

      # 3.2 安装镜像

      • 镜像选择：dreamacro/clash， 路径：https://hub.docker.com/r/dreamacro/clash

      • 端口映射：

        # 端口映射关系，本机端口:docker端口
17890:7890      # 混合代理端口
17891:7891      # socket5代理端口，可能不需要
19090:9090      # UI端口
        1
        2
        3
        4

      • 关键文件目录映射

        # 文件目录映射关系，本机目录:docker目录
/share/Container/clash/config:/root/.config/clash    # 配置文件
/share/Container/clash/ui:/ui dreamacro/clash        # UI目录
        1
        2
        3

      综上docker命令：

      docker run -d --name clash-client --restart always -p 17890:7890 -p 17891:7891 -p 19090:9090 -v /share/Container/clash/config:/root/.config/clash -v /share/Container/clash/ui:/ui dreamacro/clash
      1

      # 3.3 配置代理

      clash-client安装好了之后，就可以把配置加进去了

      sudo wget -O /share/Container/clash/config.yaml [订阅链接]
sudo wget -O /share/Container/clash/Country.mmdb https://www.sub-speeder.com/client-download/Country.mmdb
      1
      2

      打开congfig.yaml看一眼，里面内容可能不是我们想要的yml格式，比如：订阅链接可能直接返回一个base64码，这时候需要手动解析下，然后再把解析好的yaml贴到congfig.yaml里

      例如：一个没有错误的config.yaml应该是这样的

      mixed-port: 7890
allow-lan: true
bind-address: '*'
mode: rule
log-level: info
external-controller: '0.0.0.0:9090'
external-ui: '/ui'
#...后面会跟一些dns信息、代理节点信息
      1
      2
      3
      4
      5
      6
      7
      8

      保存后重启clash-client，至此clash代理就做好了

      # 3.4 控制面板（可有可无）

      推荐两个常用的UI

      1. 官方的Dashboard : https://github.com/Dreamacro/clash-dashboard/tree/gh-pages
2. 第三方的很炫酷的Dashboard: https://github.com/haishanh/yacd/tree/gh-pages
      1
      2

      打开上述路径，把gh-pages这个下面的UI文件全部down下来，放到nas的/share/Container/clash/ui目录下，这样控制面板的UI就有了

      为了能够通过UI访问并看到clash工作情况，还需要在config.yaml增加些外部访问配置

      external-controller: '0.0.0.0:9090'
#secret: '123qwe'
external-ui: '/ui'
      1
      2
      3

      保存后重启clash-client，至此clash的控制面板就做好了

      打开浏览器访问http://192.168.3.154:19090/ui，输入nas的IP地址和端口19090，就能成功访问clash了

      # 4. 客户端使用代理

      # 4.1 linux客户端

      sudo vim /etc/profile

#添加http代理、https代理、不需要代理的路径
export http_proxy="http://192.168.3.154:17890"
export https_proxy="https://192.168.3.154:17890"
export no_proxy="localhost;127.*;10.*;172.16.*;172.17.*;172.18.*;172.19.*;172.20.*;172.21.*;172.22.*;172.23.*;172.24.*;172.25.*;172.26.*;172.27.*;172.28.*;172.29.*;172.30.*;172.31.*;192.168.*"

      # 4.2 windows客户端

      "设置" --> "网络与internet" --> "代理"，手动设置代理服务器，并打开

      • 地址：192.168.3.154

      • 端口：17890

      • 不走代理的地址：localhost;127.*;10.*;172.16.*;172.17.*;172.18.*;172.19.*;172.20.*;172.21.*;172.22.*;172.23.*;172.24.*;172.25.*;172.26.*;172.27.*;172.28.*;172.29.*;172.30.*;172.31.*;192.168.*

      # 4.3 git客户端

      # 全局配置(如果要配置local，也是一样的)
git config --global http.proxy http://192.168.3.154:17890
git config --global https.proxy https://192.168.3.154:17890

# 取消配置
git config --global --unset http.proxy
git config --global --unset https.proxy

#clone代理
git clone -c http.proxy="192.168.3.154:17890" https://github.com/XXXXXXXXX.git
