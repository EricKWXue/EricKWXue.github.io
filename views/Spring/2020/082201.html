            启动流程与设计思想说明zml

            1.执行java -jar springdemo-0.0.1-SNAPSHOT.jar

            2.读取META-INF/MANIFEST.MF文件的Main-Class

            3.由默认的应用类加载器加载执行Main-Class文件，即org.springframework.boot.loader.JarLauncher

            4.JarLauncher中生成一个自定义类加载器，加载BOOT-INF在的class和jar

            5.JarLauncher读取Start-Class，即com.eric.springdemo.SpringdemoApplication，自定义类加载器加载这个class，通过反射执行其中main函数

            6.到此，整个jar包就被完全执行了。

            备注：可以使用JWDI远程调式JarLauncher的执行流程

            FAQ

            1.为什么不直接把Main-Class定义成com.eric.springdemo.SpringdemoApplication？

            com.eric.springdemo.SpringdemoApplication位于BOOT-INF/classes下，因为默认应用类加载器只能执行外层的class，若该class文件在某个文件夹下是执行不了的。

            如果把com.eric.springdemo.SpringdemoApplication直接放外层，是可以执行，但是它依赖的lib却是执行不了的。因为根据jar包打包规则，jar内不能有jar，且class外不能有莫名的文件夹，因为这些在执行jar时，都是加载不了的。若想要加载，按照旧的打包机制，得把内部依赖的jar解压缩，且class都得暴露在外面。但是一旦这样做，class文件会非常混乱，自己编写的class和依赖的class揉在一起了，且容易出现同名class冲突覆盖的情况。

            所以spring boot采用这种fat-jar的机制重新制定打包规则，且巧妙的通过自定义类记载器加载内部的依赖jar以及class。

            # 4. Spring boot 启动流程

            启动代码：new SpringApplication(primarySources).run(args)，分为两个部分

            • 创建SpringApplication对象

              • 初始化resourceLoader，资源加载器，此处为null
              • 初始化primarySources，spring boot项目启动的入口类
              • 初始化webApplicationType，应用类型，此处为web项目，即servlet
              • 初始化initializers，从META-INF/spring.factories中寻找所有实现ApplicationContextInitializer接口的实现类，加载他们的实例对象
              • 初始化listeners，从META-INF/spring.factories中寻找所有实现了ApplicationListener接口的实现类，加载他们的实例对象
              • 初始化mainApplicationClass，调用main函数启动的类

            • 调用run方法

              • 计时开始
              • 从META-INF/spring.factories中寻找所有实现SpringApplicationRunListener接口的实现类，加载他们的实例
              • ApplicationContext初始化开始，listeners.starting();
              • 准备环境environment
              • listeners.environmentPrepared(environment);
              • 打印banner
              • 创建ApplicationContext
              • 从META-INF/spring.factories中寻找所有实现SpringBootExceptionReporter接口的实现类，加载他们的实例
              • 准备ApplicationContext
              • listeners.contextPrepared(context);
              • 加载ApplicationContext
              • listeners.contextLoaded(context);
              • 刷新ApplicationContext
              • 计时结束
              • listeners.started(context);
              • 加载ApplicationRunner和CommandLineRunner
              • listeners.running(context);

            # 5. Spring boot 常用注解解析

            • @SpringBootApplication

            • @Configuration

            相当于专门用于配置bean的xml，表明这个一个配置类。成员方法上的@Bean相当于<bean>,方法会返回一个对象，这个对象要注册为Spring应用上下文中的bean

            • @EnableAutoConfiguration

            • @ComponentScan

            相当于xml配置中的<context:component-scan>，默认扫描当前路径以及子路径下所有的包，当然也可以指定扫描哪些包路径

            • @Conditional

            与接口Condition配合使用，通常应用于配置类中，符合条件才加在配置类或实例到spring容器中。衍生注解@ConditionalOnBean、@ConditionalOnProperty等等
