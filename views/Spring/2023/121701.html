      动态调整RestTemplate超时时间

      Eric Xue 2023-12-17 spring boot restTemplate

      # 1. 前言

      RestTemplate非常好用，但是在实际项目需要对不同的外部接口设置单独的超时时间。有的外部接口比较慢，超时时间设置的长一点；有的外部接口比较快，超时时间设的短一点。网上找了一下解决方案，主要有以下两种。

      1. 定义多个 RestTemplate 设置不同的超时时间。
      2. 对RestTemplate的每次请求做拦截，拦截器里修改它的超时时间。

      显然第二种实现成本较高，但会比较优雅。所以研究下它的实现！

      # 2.2 doExecute分析

      /**
 * Execute the given method on the provided URI.
 * <p>The {@link ClientHttpRequest} is processed using the {@link RequestCallback};
 * the response with the {@link ResponseExtractor}.
 * @param url the fully-expanded URL to connect to
 * @param uriTemplate the URI template that was used for creating the expanded URL
 * @param method the HTTP method to execute (GET, POST, etc.)
 * @param requestCallback object that prepares the request (can be {@code null})
 * @param responseExtractor object that extracts the return value from the response (can be {@code null})
 * @return an arbitrary object, as returned by the {@link ResponseExtractor}
 * @since 6.0
 */
@Nullable
@SuppressWarnings("try")
protected <T> T doExecute(URI url, @Nullable String uriTemplate, @Nullable HttpMethod method, @Nullable RequestCallback requestCallback,
    @Nullable ResponseExtractor<T> responseExtractor) throws RestClientException {

  Assert.notNull(url, "url is required");
  Assert.notNull(method, "HttpMethod is required");
  ClientHttpRequest request;
  try {
    request = createRequest(url, method);
  }
  catch (IOException ex) {
    ResourceAccessException exception = createResourceAccessException(url, method, ex);
    throw exception;
  }
  ClientRequestObservationContext observationContext = new ClientRequestObservationContext(request);
  observationContext.setUriTemplate(uriTemplate);
  Observation observation = ClientHttpObservationDocumentation.HTTP_CLIENT_EXCHANGES.observation(this.observationConvention,
      DEFAULT_OBSERVATION_CONVENTION, () -> observationContext, this.observationRegistry).start();
  ClientHttpResponse response = null;
  try {
    if (requestCallback != null) {
      requestCallback.doWithRequest(request);
    }
    response = request.execute();
    observationContext.setResponse(response);
    handleResponse(url, method, response);
    return (responseExtractor != null ? responseExtractor.extractData(response) : null);
  }
  catch (IOException ex) {
    ResourceAccessException exception = createResourceAccessException(url, method, ex);
    observation.error(exception);
    throw exception;
  }
  catch (RestClientException exc) {
    observation.error(exc);
    throw exc;
  }
  finally {
    if (response != null) {
      response.close();
    }
    observation.stop();
  }
	}

      # 2.3 createRequest分析

      RestTemplate 中的 Request 都是由 RequestFactory 完成创建的。

      /**
 * Create a new {@link ClientHttpRequest} via this template's {@link ClientHttpRequestFactory}.
 * @param url the URL to connect to
 * @param method the HTTP method to execute (GET, POST, etc)
 * @return the created request
 * @throws IOException in case of I/O errors
 * @see #getRequestFactory()
 * @see ClientHttpRequestFactory#createRequest(URI, HttpMethod)
 */
protected ClientHttpRequest createRequest(URI url, HttpMethod method) throws IOException {
  ClientHttpRequest request = getRequestFactory().createRequest(url, method);
  initialize(request);
  if (logger.isDebugEnabled()) {
    logger.debug("HTTP " + method.name() + " " + url);
  }
  return request;
}

      getRequestFactory()默认情况会得到 SimpleClientHttpRequestFactory。我们可以在这个类中看到 setConnectTimeout、setReadTimeout 方法，因此可以初步判断超时时间的设定可能只要在这个RequestFactory身上下功夫就可以了。

      再看一下SimpleClientHttpRequestFactory，找到设置超时时间的方法

      /**
 * Template method for preparing the given {@link HttpURLConnection}.
 * <p>The default implementation prepares the connection for input and output, and sets the HTTP method.
 * @param connection the connection to prepare
 * @param httpMethod the HTTP request method ({@code GET}, {@code POST}, etc.)
 * @throws IOException in case of I/O errors
 */
protected void prepareConnection(HttpURLConnection connection, String httpMethod) throws IOException {
  if (this.connectTimeout >= 0) {
    connection.setConnectTimeout(this.connectTimeout);
  }
  if (this.readTimeout >= 0) {
    connection.setReadTimeout(this.readTimeout);
  }

  boolean mayWrite =
      ("POST".equals(httpMethod) || "PUT".equals(httpMethod) ||
          "PATCH".equals(httpMethod) || "DELETE".equals(httpMethod));

  connection.setDoInput(true);
  connection.setInstanceFollowRedirects("GET".equals(httpMethod));
  connection.setDoOutput(mayWrite);
  connection.setRequestMethod(httpMethod);
}

      # 3.2 createRequest分析

      我们看一下HttpComponentsClientHttpRequestFactory的createRequest方法，再研究如何去设置超时时间

      @Override
public ClientHttpRequest createRequest(URI uri, HttpMethod httpMethod) throws IOException {
  HttpClient client = getHttpClient();

  ClassicHttpRequest httpRequest = createHttpUriRequest(httpMethod, uri);
  postProcessHttpRequest(httpRequest);
  HttpContext context = createHttpContext(httpMethod, uri);
  if (context == null) {
    context = HttpClientContext.create();
  }

  // Request configuration not set in the context
  if (context.getAttribute(HttpClientContext.REQUEST_CONFIG) == null) {
    // Use request configuration given by the user, when available
    RequestConfig config = null;
    if (httpRequest instanceof Configurable configurable) {
      config = configurable.getConfig();
    }
    if (config == null) {
      config = createRequestConfig(client);
    }
    if (config != null) {
      context.setAttribute(HttpClientContext.REQUEST_CONFIG, config);
    }
  }

  if (this.bufferRequestBody) {
    return new HttpComponentsClientHttpRequest(client, httpRequest, context);
  }
  else {
    return new HttpComponentsStreamingClientHttpRequest(client, httpRequest, context);
  }
}

      1. 重写createHttpContext方法，直接生成一个新的带有RequestConfig的HttpContext。
      2. 重写createRequestConfig方法。>> 这个不太保险，该方法太深入了，万一走不到此处就完犊子了，因此不推荐。

      # 3.3 改造实现

      1. 请求前，将超时时间存储在ThreadLocal中。
      2. 自定义CustomHttpComponentsClientHttpRequestFactory继承HttpComponentsClientHttpRequestFactory，重写createHttpContext方法：读取ThreadLocal中的超时时间，生成RequestConfig塞入超时时间，再塞入HttpContext中返回。
      3. 请求后，清除ThreadLocal的超时时间
      /**
 * 定义超时时间对象
 */
@Data
@AllArgsConstructor
@NoArgsConstructor
public class HttpTimeoutBean {
    private Timeout readTimeout;
}

/**
 * http配置上下文
 */
public class HttpConfigHolder {
    private static final ThreadLocal<HttpTimeoutBean> THREAD_LOCAL = new NamedThreadLocal<>("http请求超时时间切换");
		//塞入时间
    public static void bind(HttpTimeoutBean httpTimeout){
        THREAD_LOCAL.set(httpTimeout);
    }
		//获取时间
    public static HttpTimeoutBean peek(){
        return THREAD_LOCAL.get();
    }
		//清除时间
    public static void unbind(){
        THREAD_LOCAL.remove();
    }
}

/**
 * http client配置
 */
@Configuration
public class HttpRequestConfig {

    @Bean
    public RestTemplate createRestTemplate(){
        CustomHttpComponentsClientHttpRequestFactory factory = new CustomHttpComponentsClientHttpRequestFactory(HttpConfigHolder::peek);
        return new RestTemplate(factory);
    }

    private class CustomHttpComponentsClientHttpRequestFactory extends HttpComponentsClientHttpRequestFactory {
      
        @Nullable
        private Supplier<HttpTimeoutBean> httpTimeoutFactory;

        public CustomHttpComponentsClientHttpRequestFactory(@Nullable Supplier<HttpTimeoutBean> httpTimeoutFactory) {
            super();
            this.httpTimeoutFactory = httpTimeoutFactory;
        }

        @Override
        protected HttpContext createHttpContext(HttpMethod httpMethod, URI uri) {
            HttpClientContext context = HttpClientContext.create();
            RequestConfig config = super.createRequestConfig(getHttpClient());
            HttpTimeoutBean httpTimeout = (this.httpTimeoutFactory != null ? this.httpTimeoutFactory.get() : null);
            if(httpTimeout != null){
                config = RequestConfig.custom()
                        .setResponseTimeout(httpTimeout.getReadTimeout())
                        .build();
            }
            context.setRequestConfig(config);
            return context;
        }
    }
}

      # 3.4 升级思路

      因为每次使用前，都要HttpConfigHolder.bind和HttpConfigHolder.unbind，导致每次都要try...finally...。繁琐，且代码看上去会非常不干净！

      因此想用自定义注解配合Spring AOP写个切面，自动完成以上操作。让service层回归业务！

      先添加AOP的依赖

      <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-aop</artifactId>
</dependency>
      1
      2
      3
      4

      大概思路，在原先基础上加两个东西：

      1. 定义超时时间的注解
      2. 实现环切，作用于每个加了超时时间注解的业务方法上
      @Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
public @interface HttpTimeout {
    /**
     * read time out, 单位：秒
     * @return
     */
    int readTimeout() default -1;
}


@Aspect
@Configuration
public class HttpTimeoutAspect {

    /**
     * 注解写法：被标记HttpTimeout的方法都纳入切面
     * 表达式写法：类路径
     */
    @Pointcut("@annotation(com.ericx.springboottest.annotation.HttpTimeout)")
    public void pointcut() {
    }
    /**
     * 环绕通知
     */
    @Around("@annotation(httpTimeout)")
    public Object around(ProceedingJoinPoint joinPoint, HttpTimeout httpTimeout) throws Throwable {
        HttpTimeoutBean bean = new HttpTimeoutBean();
        bean.setReadTimeout(Timeout.ofSeconds(httpTimeout.readTimeout()));
        HttpConfigHolder.bind(bean);
        try {
            return joinPoint.proceed();
        } catch (Throwable e) {
            throw e;
        }finally {
            HttpConfigHolder.unbind();
        }
    }
}
