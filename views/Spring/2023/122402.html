      线上内存溢出问题排查

      Eric Xue 2023-12-24 spring boot actuator

      # 前言

      现象：Rest服务重启过于频繁，从Grafana上看每次都是内存飙升引起的服务重启。

      背景：目前K8S容器内存阈值是75%，超过了就会主动把服务kill掉再重新启动，无法适用JVM参数-XX:HeapDumpOnOutOfMemoryError-XX:HeapDumpPath。然而生产环境装的又是JRE，没有jmap等工具，无法配置重启自动dump内存镜像。

      # 1. 手写代码取内存镜像

      需要actuator这个包

      <dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

      使用org.springframework.boot.actuate.management.HeapDumpWebEndpoint这个类可以直接dump当前内存镜像。

      @SneakyThrows
@Override
public void dumpMemory() {
    File file = new File(storagePath + "/rest.hprof");
    //只保留第一个dump记录
    if(file.exists()){
        return;
    }
    HeapDumpWebEndpoint heapDumpWebEndpoint = new HeapDumpWebEndpoint();
    WebEndpointResponse<Resource> resourceWebEndpointResponse = heapDumpWebEndpoint.heapDump(true);
    Resource body = resourceWebEndpointResponse.getBody();
    FileCopyUtils.copy(body.getInputStream(), Files.newOutputStream(file.toPath()));
}

      点进去看了一下，内部是通过反射的方式，运行了com.sun.management.HotSpotDiagnosticMXBean的dumpHeap方法。

      最后写个定时器监控内存，每10s监控一次。目前关于内存的启动配置：-Xms最大内存的75% -Xmx最大内存的75%

      @Scheduled(cron = "0/10 * * * * ?")
public void scheduleTask(){
    //按照现在的配置，totalMemory == maxMemory
    long heapSize = Runtime.getRuntime().totalMemory();
    long heapMaxSize = Runtime.getRuntime().maxMemory();
    //空闲内存
    long heapFreeSize = Runtime.getRuntime().freeMemory();
    //如果rest 目前佔用的memory 已經佔用超過了85%，那麼觸發一次dump
    if(heapFreeSize < heapMaxSize * 0.15){
        //dump文件
        dumpMemory();
        //發郵件給我
        sendMail();
    }
}

      # 2.2 导入hprof文件

      堆的总大小为1.6G，其中1.3G被某个线程占了，很明显问题出在这里。

      不想自己去看图表（Hostogram、Dominator Tree）的话，可以直接点Leak Suspects看eclipse生成的报表

      org.apache.ibatis.executor.result.DefaultResultHandler内部持有一个List，从这个类基本可以看出是与数据库的查询相关。这个List中的元素共占了1.3G，初步可以判断出是在一次查询中从数据库中一次查询出了太多数据，造成了内存溢出。
